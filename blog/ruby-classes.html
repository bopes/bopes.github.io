Ruby Classes: How and Why

Feb 29 2016

Arrays and Hashes are great for storing individual sets of data, but what if you need to store different versions of the same thing? For instance, if you want to track all of your favorite movies, you could create an individual hash for each containing the title, year, director, and lead actor, but that would take forever. Instead, you can use a Class to store this information, which gives you the access of a hash but also the flexibility to create your own.

To create a Class, you first need to define it with its initial arguments:

class Movie
initialize(title,year,director,actor)
end
end

This means that we have now created a new class called "Movie", and that whenever we log a new movie, it will have a title, year, director and actor. To make these data points generally available for use within the class, we will define them as instance variables:

class Movie
initialize(title,year,director,actor)
@title = title
@year = year
@director = director
@actor = actor
end
end

These 'instance variables', determined with the prefix @, are variables that can be at any time within the class, regardless of which movie (or instance of the class) you are using. These variables are not available outside of the class on their own, meaning other methods cannot find them, without a reader or writer method such as attr_reader, attr_writer, or attr_accessor, shown here:

class Movie
attr_reader :title, :year, :director, :actor
initialize(title,year,director,actor)
@title = title
@year = year
@director = director
@actor = actor
end
end

Now these instance variables can be called by other methods outside of the class.

You can also create methods specific to this class. For instance, if I wanted to create a shorthand way to view the title, year, and director for any given movie, I can create an instance method within the Movie class that will only work for my movies. But it will still work for any movie I put into my class. Check it out:

class Movie
attr_reader :title, :year, :director, :actor
initialize(title,year,director,actor)
@title = title
@year = year
@director = director
@actor = actor
end
def overview
puts "#{@title}, #{year}. Directed by {@director}. Stars {@actor}."
end
end

This new overview method will now be available for any movies I put into this class, or any movies I enter in the future.

Classes are a great way to store multiple versions of the same thing, where you can define the precise methods and variables types you will need. You only have to set the class up once, and it will be very simple to add new instances over time without having to recode or define the same structure or methods over again.
