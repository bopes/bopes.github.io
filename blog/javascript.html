Looping

Mar 7 2016

When you're dealing with arrays or hashes in Ruby, you frequently need to apply some code to every element in the container. Ruby makes it easy, because the enumerable methods like #each, #map, and #collect know how to go through each element and do something. But say you aren't using Ruby -- you're using JavaScript, which doesn't have any enumerable methods. So how do you affect every item in an array?

Fortunately, it's still possible. It just requires you to use a loop, which does a bit of code over and over again, that's tracked by manual counting (For the record, you can use a manual counter in Ruby as well. You just don't have to).

The main looping mechanism in JavaScript is the FOR loop. To execute a FOR loop, you need to set up the 'looper,' and then put your code inside of it.

The 'looper', as I call it, specifies these three things:

  1. The Counter

    This is the variable that keeps track of how many times you've looped, and its starting value.

    Example: var counter = 0
  2. The Loop Criteria

    This is the criteria that determines how many times you loop. As long as this criteria evaluates to "true", the looper will run again. Frequently, this sets the upper bound on the counter.

    Example: counter < 5
  3. The Counter Increase

    This is actually optional, but I always like to include it. It determines what changes with each successive loop.

    Example: counter = counter + 1

    (Sidenote - this particular instance can be shortened to counter += 1, to increase by 1 with each loop. It can be shortened even further to counter++, if brevity is really your thing!)
When you put it all together, it looks like this:

for ( var counter = 0 ; counter < 5 ; counter ++ ) {....code block....} ;

This loop will do whatever appears in the code block exactly five times.

If you want to execute some code against every element in an array, you can do the following:

for ( var counter = 0 ; counter < array.length ; counter ++ ) { array[counter] ...... } ;

This is a great way to loop through an entire array, even if you don't know how long the array is going to be!

The way to do this same loop in Ruby would be the following:

array.each do
some code
end

As you can see, Ruby' #each method makes it a lot easier to read and write these types of loops. But that just shows how readable Ruby can be -- JavaScript can still execute the exact same functions, it just takes a little manual manipulating to get it working!
