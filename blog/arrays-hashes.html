Arrays vs Hashes: When to use each

Feb 14 2016

Ruby makes it easy to define a variable so you can access it a later date. For instance, you can set up a local variable:

favorite_color = "Blue"

and have access to that value at any time in the future.

But what if you want to store a list? Say you want to store your shopping list, with milk, eggs, and bread. You could save it as:

shopping_list = "Milk, eggs, bread"

but that would get cumbersome quickly, and you wouldn't be able to remove indvidual items easily.

Instead, you could save it as an array or a hash.

An array is a list of objects, arranged in a specific order. So our example would look like this:

shopping_list = ["Milk","Eggs","Bread"]

With an array, each item is labeled according to its order, beginning with 0. So in our example, "Milk" is the 0th item, "Eggs" is the 1st item, and "Bread" is the second item. If you want to refer to a specific item, you can specify the specifict item in the array. So:

shopping_list[0] = "Milk"
shopping_list[1] = "Eggs"
shopping_list[2] = "Bread"

This makes is easier to remove one item from this list after you buy it, without affecting the other items. You can delete "Eggs" by removing that specific items from the list.

Arrays are great for lists where you care about what items are included.

But sometimes you care about more than that. Say you are sending a thank you note to your best friend. You need their mailing address, which includes their street address, apartment number, city, state, and zip code. To store all this information, you would use a hash.

A hash is a list of data point, organized into key-value pairs. In other words, a hash lets you establish what data points you need (called keys), and then assign a value to each of of those points. So we could create a hash like this:

best_friend = { address => "1234 Main St", city => "Austin", state => "TX", zip => "98765" }

This may seem like more work, but it lets you look up specific details really easily. Instead of remembering that "City" is fourth on the list, you can just look up the "City" key and find the associated value, like this:

best_friend["city"] = "Austin"

This also makes it easier for other people to understand and use your data, since they can just read it for themselves.
